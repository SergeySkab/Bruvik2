\# SYSTEM INSTRUCTION FOR AI DEVELOPMENT AGENT

\*\*Context:\*\* You are an Expert Front-End Developer and UI/UX Designer. Your task is to generate a fully responsive, multi-page web project based on the provided historical and marketing document about the village of Bruvik.   
\*\*Goal:\*\* Create a high-end "longread" style tourism website focused on immersive storytelling, large typography, and full-screen photography.

\#\# 1\. БИЗНЕС-ТРЕБОВАНИЯ (Business Requirements Document \- BRD)

\#\#\# 1.1. Цель проекта  
Создание цифровой платформы для территориального брендинга деревни Брувик (Норвегия), трансформирующей исторические факты в коммерческий сторителлинг.

\#\#\# 1.2. Уникальное ценностное предложение (UVP)  
Главный слоган сайта, который должен быть интегрирован на главной странице: \*"Брувик: Там, где дикая природа испытывает тело на прочность, а суровая история — углубляет душу"\*.

\#\#\# 1.3. Целевая аудитория  
\* Любители активного отдыха (хайкинг на Брувикнипа).  
\* Исследователи истории архитектуры (Bruvik kyrkje).  
\* Поклонники "тёмного туризма" и скандинавского нуара (остров Ульфснесёй).

\---

\#\# 2\. ТЕХНИЧЕСКОЕ ЗАДАНИЕ (Software Requirements Specification \- SRS)

\#\#\# 2.1. Технологический стек  
\* \*\*HTML5:\*\* Семантическая разметка (использование \`\<header\>\`, \`\<main\>\`, \`\<article\>\`, \`\<section\>\`, \`\<aside\>\`, \`\<footer\>\`).  
\* \*\*CSS3:\*\* Flexbox/Grid для лейаутов. Использование CSS-переменных (Custom Properties) для цветовой схемы и типографики. Мобильная адаптивность (Mobile First) через \`@media\` запросы.  
\* \*\*JavaScript (Vanilla):\*\* Минимальное использование для интерактивности (плавная прокрутка, бургер-меню для мобильных устройств, анимация появления контента при скролле).  
\* \*\*Сборка:\*\* Структура проекта должна работать статично (без сложных фреймворков вроде React/Vue), чтобы сайт можно было легко хостить на GitHub Pages или Vercel.

\#\#\# 2.2. Требования к UI/UX (Дизайн-система "Лонгрид")  
\* \*\*Типографика:\*\* Оптимизирована для длинных текстов. Ширина текстового контейнера (читаемой области) не должна превышать \`max-width: 750px;\` или \`800px;\`. Межстрочный интервал (line-height) для основного текста — \`1.6\` или \`1.8\`.  
\* \*\*Изображения (Hero Sections):\*\* Полноэкранные блоки (100vh). Фотографии должны загружаться с использованием CSS-свойства \`object-fit: cover\` (или \`background-size: cover\`).  
\* \*\*Читаемость:\*\* Поверх полноэкранных фотографий, где есть текст, ОБЯЗАТЕЛЬНО использовать CSS \`linear-gradient\` (overlay) с черным/темным цветом и прозрачностью \`0.5 \- 0.7\` для обеспечения контрастности белого текста.  
\* \*\*Цветовая палитра:\*\* Скандинавский минимализм. Темные, глубокие тона (сланец, темно-синий, хвойный) для фонов блоков "Темного туризма", и светлые, "воздушные" тона для блоков о природе.

\#\# 4\. ТРЕБОВАНИЯ К SEO (On-Page SEO Criteria)

При генерации HTML-кода агент обязан использовать семантическое ядро из Таблицы 6.4 документа:  
\* Использовать ключевые слова (например, "Bruvik kyrkje historie", "Bruviknipa fottur Osterøy") в тегах \`\<title\>\` соответствующих страниц.  
\* Использовать эти же ключи в заголовках \`\<h1\>\`, \`\<h2\>\`.  
\* Добавлять заглушки для картинок в виде \`\<img src="placeholder.jpg" alt="\[SEO ключевое слово\]"\>\` для последующей замены на реальные фото.
